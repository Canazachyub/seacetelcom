¿Qué es un identificador de proceso de contratación (OCID)?
Todo proceso de contratación se identifica de forma única y global mediante un identificador de proceso de contratación (OCID, por sus siglas en inglés) .

Se compone de un prefijo OCID y un identificador para el proceso de contratación.

Prefijo OCID: Es emitido a solicitud por la Open Contracting Partnership. Está conformado por el prefijo ‘ocds’ seguido de una cadena de seis caracteres alfanuméricos. El prefijo OCID es único y sensible a las mayúsculas y minúsculas.
Identificador para el proceso de contrataciones: Identificador interno utilizado por el publicador para identificar de forma única cada proceso de contratación.
Ejemplo de un identificador de proceso de contratación (OCID):
ocds-dgv273-2022-1877-101

¿Qué es un entrega, (release)?
Toda nueva información sobre un proceso de contratación se describe dentro de una entrega (release). Cada entrega tiene un OCID.
Las entregas (releases) son inmutables donde cada entrega presenta información sobre un evento particular en la vida de un proceso de contratación. Los publicadores no deben editar una entrega después de su publicación; una nueva entrega puede crearse definiendo un nuevo identificador único (id), la fecha (date) y el identificador de proceso de contratación (OCID) de la entrega.

Ejemplo de un identificador único (id) de una entrega:
ocds-dgv273-2022-1877-101-2022-11-15T10:42:04.968512-05:00

¿Qué es un paquete de entregas (release package)?
Toda entrega (release) se publica dentro de un paquete de entregas (release package).

El paquete de entregas proporciona metadatos sobre cada entrega que contiene:

La URI (uri) del paquete de entregas.
La versión (version) del OCDS usada.
Una lista de extensiones (extensions) del OCDS usadas.
La fecha de publicación (publishedDate) del paquete de entregas.
Una lista de una o más entregas (releases).
El publicador (publisher) del paquete de entregas.

¿Qué es un registro, (record)?
Aunque pueden haber múltiples entregas (releases) sobre un proceso de contratación, debe haber un solo registro (record) por proceso de contratación, donde se unen todas las entregas disponibles para el proceso de contratación.

Un registro (record) contiene:

Un identificador único del proceso de contratación (OCID)
Una lista de entregas (releases) sobre el proceso de contratación, puede ser dando la ubicación (url) del paquete de entregas (release package) y la fecha (date) de la entrega, o incorporando una copia completa de la entrega.
Una entrega compilada (compiledRelease): es la última versión de todos los datos del proceso de contratación.
Esto permite al registro funcionar como un índice de todas las entregas sobre un proceso de contratación.

¿Qué es un paquete de registros (record package)?
Todo registro (record) se publica dentro de un paquete de registros (record package).
El paquete de registros proporciona metadatos sobre cada registro que contiene:

La URI (uri) del paquete de entregas.
La versión (version) del OCDS usada.
Una lista de extensiones (extensions) del OCDS usadas.
La fecha de publicación (publishedDate) del paquete de entregas.
Una lista de una o más registros (records).
El publicador (publisher) del paquete de entregas.


¿Cómo probar un endpoint?
Expandir una categoría, ya sea de Releases (Entregas), Records (Registros) o Files (Archivos de descarga masiva).
Seleccionar y expandir un método.
Seleccionar la opción Try it out para poder modificar los parámetros.
Hacer clic en execute para obtener los resultados.
